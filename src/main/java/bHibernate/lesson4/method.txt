1. mappedBy
        - объявляет сторону, которая не несет отвественность за отношения

2. optional - true/flase
        - показывает является ли значение в этом поле обязательным или нет

3.fetch
        - FetchType.EAGER
            - при загрузке владеемого объекта необходимо сразу загрузить и коллекцию владельцев.
            - в памяти будут находиться все загруженные объекты, даже если нужен только один объект из всех.
        - FetchType.LAZY
            - связанные объекты загружаются только по мере необходимости, т. е при обращении
            - при этом требуется, чтобы соединение с базой (или транзакция) сохранялись (объект должен быть attached)
            - поэтому для работы с lazy объектами тратится больше ресурсов на поддержку соединений

4.CascadeType
    - определяет влияние различных операций одной сущности на связанные
        - ALL       - все операции родителя будут отражены на ребенке (save, delete, update, evict, lock, replicate, merge, persist);
        - REFRESH   - обновление сущности
        - REMOVE    - удаление сущности
        - MERGE     -
        - PERSIST   -
        - DETACH    - отсоединения сущности

5.orphanRemoval - true/flase
        - помечает «дочернюю» сущность, которая будет удалена, когда на нее больше нет ссылок из «родительской» сущности
          (например, когда вы удаляете дочернюю сущность из соответствующей коллекции родительской сущности)
